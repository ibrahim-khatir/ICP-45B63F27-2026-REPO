# Static Website Hosting on AWS

## Project Overview

This project demonstrates hosting a static website on AWS using Amazon S3 and Amazon CloudFront while following AWS security best practices. The S3 bucket remains private, and CloudFront securely accesses the content through Origin Access Control (OAC).

---

## Bucket Creation

### Bucket Name
intern-static-website

### Region
(Your AWS Region)

### Configuration

- Created an Amazon S3 bucket.
- Block Public Access remained enabled.
- Uploaded:
  - index.html
  - style.css
  - assets/profile.jpg
- Static Website Hosting was not enabled because the website is served through CloudFront.

---

## CloudFront Configuration

Created a CloudFront distribution with the following configuration:

- Origin: Amazon S3 bucket
- Viewer Protocol Policy: Redirect HTTP to HTTPS
- Default Root Object: index.html
- Cache Policy: Managed-CachingOptimized
- Distribution Status: Deployed

---

## Origin Access Control (OAC)

Configured Origin Access Control to allow CloudFront to securely access the private S3 bucket.

Steps:

1. Created a new Origin Access Control.
2. Attached the OAC to the S3 origin.
3. Updated the S3 bucket policy using the policy generated by CloudFront.
4. Verified that CloudFront could retrieve objects from the bucket.

---

## Problems Encountered

### Problem 1

Received **403 Access Denied** when opening the CloudFront URL.

### Cause

CloudFront did not initially have permission to access the private S3 bucket.

### Solution

- Verified the bucket policy.
- Corrected the CloudFront origin configuration.
- Ensured the correct Origin Access Control was attached.
- Set the Default Root Object to index.html.
- Waited for CloudFront deployment to complete.
